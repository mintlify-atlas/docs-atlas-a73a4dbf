## Overview

Analyze blast radius of changing an artifact. Uses BFS by depth to classify impact: depth 1 = WILL_BREAK, depth 2 = LIKELY_AFFECTED, depth 3 = MAY_NEED_REVIEW. When code intelligence is available, uses call graph for deeper analysis.

This tool helps you understand the consequences of modifying a requirement, use case, or any other artifact by traversing the traceability graph and identifying all dependent or dependency artifacts.

## Parameters

<ParamField path="artifact_id" type="string" required>
  The artifact ID to analyze (e.g. `REQ-AUTH-001`, `UC-003`)
</ParamField>

<ParamField path="direction" type="enum" required>
  Direction to traverse:
  - `upstream` - What does this depend on?
  - `downstream` - What depends on this?
</ParamField>

<ParamField path="maxDepth" type="number" default="3">
  Maximum traversal depth
</ParamField>

## Impact depth labels

- **Depth 1 (WILL_BREAK)**: Direct dependencies that will definitely be affected
- **Depth 2 (LIKELY_AFFECTED)**: Second-degree connections that will likely need updates
- **Depth 3 (MAY_NEED_REVIEW)**: Third-degree connections that may need review

## Examples

### Analyze downstream impact of a requirement

```json
{
  "artifact_id": "REQ-AUTH-001",
  "direction": "downstream"
}
```

### Check upstream dependencies with limited depth

```json
{
  "artifact_id": "UC-LOGIN-003",
  "direction": "upstream",
  "maxDepth": 2
}
```

### Assess blast radius before major changes

```json
{
  "artifact_id": "API-USER-MGMT",
  "direction": "downstream",
  "maxDepth": 4
}
```
